Instrukcja podnoszenia wersji

System Zlecenia v.2 - Zarządzanie numerami wersji aplikacji

1. Wprowadzenie

System wersjonowania aplikacji Zlecenia v.2 pozwala na śledzenie zmian i aktualizacji wprowadzanych w aplikacji. Numery wersji są wyświetlane w stopce strony logowania i mają format:

v.X.Y.ZZZ

gdzie:

  • X - numer wersji głównej (major) - zwiększany przy dużych zmianach, które mogą wpływać na kompatybilność
  • Y - numer wersji pomniejszej (minor) - zwiększany przy dodawaniu nowych funkcji
  • ZZZ - numer łatki/poprawki (patch) - zwiększany przy drobnych poprawkach i ulepszeniach, zawsze trzycyfrowy

2. Jak zmienić numer wersji

2.1. Użycie skryptu wersja.js

W projekcie zaimplementowano skrypt wersja.js, który automatyzuje proces zmiany wersji. Skrypt znajduje się w katalogu scripts.

2.2. Dostępne komendy

Zwiększenie numeru poprawki (patch)

Użyj tej komendy dla drobnych poprawek i ulepszeń (np. 2.0.001 → 2.0.002):

node scripts/wersja.js patch

Zwiększenie numeru wersji pomniejszej (minor)

Użyj tej komendy podczas dodawania nowych funkcji, które nie wpływają na kompatybilność (np. 2.0.001 → 2.1.000):

node scripts/wersja.js minor

Zwiększenie numeru wersji głównej (major)

Użyj tej komendy przy dużych zmianach, które mogą wpływać na kompatybilność (np. 2.0.001 → 3.0.000):

node scripts/wersja.js major

Ustawienie konkretnego numeru wersji

Aby ręcznie ustawić konkretny numer wersji:

node scripts/wersja.js set 2.5.123
Uwaga: Po zmianie numeru wersji należy zrestartować serwer, aby zmiany były widoczne na stronie logowania.

3. Gdzie wersja jest przechowywana

Numer wersji jest przechowywany w pliku konfiguracyjnym shared/config.ts. W pliku tym znajdują się trzy zmienne:

  • major - numer wersji głównej
  • minor - numer wersji pomniejszej
  • patch - numer łatki/poprawki

4. Najlepsze praktyki wersjonowania

  • Zwiększaj numer patch przy poprawkach błędów i drobnych ulepszeniach, które nie dodają nowych funkcji.
  • Zwiększaj numer minor przy dodawaniu nowych funkcji, które nie wpływają na istniejącą funkcjonalność.
  • Zwiększaj numer major przy dużych zmianach, które mogą wpływać na istniejącą funkcjonalność.
  • Aktualizuj dokumentację i informacje dla użytkowników przy wprowadzaniu znaczących zmian.
  • Utrzymuj dziennik zmian (changelog), aby śledzić, co zostało zmienione w każdej wersji.
Ważne: Nie edytuj ręcznie pliku shared/config.ts, chyba że jest to absolutnie konieczne. Zawsze używaj skryptu wersja.js, aby uniknąć błędów.

5. Kod konfiguracyjny

Plik konfiguracyjny shared/config.ts zawiera następującą strukturę:

// Plik konfiguracyjny aplikacji

// Ustawienia wersji aplikacji
export const APP_VERSION = {
  major: 2,  // Duże zmiany w funkcjonalności
  minor: 0,  // Dodanie nowych funkcji
  patch: 2,  // Poprawki błędów, drobne zmiany
};

// Funkcja generująca numer wersji w formacie string
export function getVersionString(): string {
  return `v.${APP_VERSION.major}.${APP_VERSION.minor}.${String(APP_VERSION.patch).padStart(3, '0')}`;
}

// Inne ustawienia konfiguracyjne aplikacji
export const APP_CONFIG = {
  name: 'Zlecenia',
  copyright: '2025',
  versionString: getVersionString(),
};
