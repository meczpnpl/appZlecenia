Instrukcja korzystania z GitHub w Replit Shell

Ważne: Replit może ograniczać niektóre operacje Git ze względów bezpieczeństwa. W przypadku napotkania ograniczeń, skontaktuj się z obsługą Replit.

1. Konfiguracja początkowa

Konfiguracja użytkownika Git

Ustaw swoją tożsamość dla commitów:

git config --global user.name "Twoje Imię"
git config --global user.email "twoj.email@przyklad.pl"

Inicjalizacja repozytorium

Jeśli zaczynasz nowy projekt:

git init

Jeśli masz problemy z blokadą plików:

rm -f .git/index.lock
rm -f .git/config.lock

Łączenie z istniejącym repozytorium GitHub

git remote add origin https://github.com/twojaNazwaUzytkownika/nazwa-repozytorium.git

2. Codzienna praca

Sprawdzanie statusu

Zobacz, które pliki zostały zmienione:

git status

Dodawanie plików do śledzenia

Dodaj wszystkie pliki:

git add .

Dodaj konkretne pliki:

git add ścieżka/do/pliku.js ścieżka/do/innego/pliku.css

Tworzenie commita

Zapisuje aktualny stan jako punkt w historii:

git commit -am "Opis wprowadzonych zmian"

Wysyłanie zmian na GitHub

git push origin main

Przy pierwszym wysyłaniu możesz potrzebować:

git push -u origin main

Podczas push zostaniesz poproszony o poświadczenia GitHub. Użyj token PAT jako hasło.

Pobieranie zmian z GitHub

git pull origin main

3. Zarządzanie wersjami

Wyświetlanie historii commitów

git log

Skrócona wersja z jedną linią na commit:

git log --oneline

Przywracanie do wcześniejszej wersji

Soft reset (zachowuje zmiany jako niezatwierdzone):

git reset --soft HASH_COMMITA

Hard reset (całkowicie usuwa zmiany):

git reset --hard HASH_COMMITA

Gdzie HASH_COMMITA to identyfikator commita z git log, np. a1b2c3d

Przywracanie konkretnego pliku

git checkout HASH_COMMITA -- ścieżka/do/pliku.js

4. Praca z gałęziami

Tworzenie nowej gałęzi

git checkout -b nazwa-nowej-galezi

Przełączanie między gałęziami

git checkout nazwa-galezi

Łączenie gałęzi

# Najpierw przejdź do gałęzi docelowej
git checkout main
# Następnie połącz zmiany z innej gałęzi
git merge nazwa-galezi

5. Rozwiązywanie problemów

Usuwanie plików blokady

rm -f .git/index.lock
rm -f .git/config.lock

Resetowanie nieśledzonych plików

git clean -fd
Uwaga: Ta komenda usuwa wszystkie nieśledzone pliki i katalogi! Używaj ostrożnie.

Tworzenie pliku .gitignore

Aby ignorować określone pliki lub foldery:

echo "nazwa_pliku.txt" > .gitignore
echo "folder/" >> .gitignore
echo "*.log" >> .gitignore

6. Przykłady typowych scenariuszy

Zadanie Komendy
Utworzenie punktu kontrolnego przed zmianami git add .
git commit -m "Punkt kontrolny przed zmianami"
Cofnięcie do poprzedniego commita git log --oneline (znajdź hash commita)
git reset --hard HASH_COMMITA
Przywrócenie pojedynczego pliku git checkout HASH_COMMITA -- ścieżka/do/pliku.js
git commit -m "Przywrócono plik z poprzedniej wersji"
Synchronizacja z GitHub po zmianach git pull origin main
git add .
git commit -m "Moje zmiany"
git push origin main
Pamiętaj: Wykonuj regularne commity, aby mieć punkty kontrolne, do których możesz wrócić. Dobra praktyka to commit po każdej logicznej zmianie w kodzie.
