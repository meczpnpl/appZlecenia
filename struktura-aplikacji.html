Struktura Aplikacji Bel-Pol

Poniżej przedstawiona jest pełna struktura katalogów i plików aplikacji wraz z opisami najważniejszych elementów.

Główne foldery projektu

client Kod źródłowy frontendu aplikacji (React)
public Pliki statyczne dostępne publicznie
src Kod źródłowy aplikacji frontendowej
components Komponenty React używane w aplikacji
orders Komponenty specyficzne dla zleceń
permissions Komponenty do zarządzania uprawnieniami
permission-gate.tsx Komponent do warunkowego renderowania elementów na podstawie uprawnień
permission-guard.tsx Komponent do ochrony całych stron/widoków na podstawie uprawnień
ui Podstawowe komponenty UI
hooks Hooki React
use-permissions.ts Hook do sprawdzania uprawnień użytkownika
use-toast.ts Hook do wyświetlania powiadomień
use-auth.ts Hook do obsługi autoryzacji
lib Biblioteki i funkcje pomocnicze
queryClient.ts Konfiguracja React Query
apiRequest.ts Funkcja do wykonywania zapytań API
auth.ts Logika autoryzacji
middleware Middleware dla aplikacji klienckiej
pages Komponenty stron/podstron aplikacji
Orders.tsx Strona z listą zleceń
OrderDetails.tsx Strona ze szczegółami zlecenia
ViewPermissionsManager.tsx Panel zarządzania uprawnieniami
Dashboard.tsx Strona główna/dashboard
Login.tsx Strona logowania
utils Funkcje pomocnicze
App.tsx Główny komponent aplikacji
main.tsx Punkt wejściowy aplikacji React
index.css Główny plik CSS
permissions-fix.js Skrypt naprawy systemu uprawnień
.env Zmienne środowiskowe dla frontendu
index.html Główny plik HTML aplikacji
server Kod źródłowy backendu aplikacji (Express.js)
routes Definicje tras API
permissions.ts Endpointy zarządzania uprawnieniami
orders.ts Endpointy zarządzania zleceniami
auth.ts Endpointy autoryzacji
users.ts Endpointy zarządzania użytkownikami
services Usługi biznesowe
permission-service.ts Usługa zarządzania uprawnieniami
order-service.ts Usługa zarządzania zleceniami
notification-service.ts Usługa powiadomień
middleware Middleware dla API
auth-middleware.ts Middleware do autoryzacji
permission-middleware.ts Middleware do sprawdzania uprawnień
index.ts Główny plik serwera
database-storage.ts Warstwa dostępu do bazy danych
db.ts Konfiguracja połączenia z bazą danych
routes.ts Główny plik konfiguracji tras API
shared Pliki współdzielone między frontendem i backendem
schema.ts Definicje schematów bazy danych (Drizzle ORM)
permission-schema.ts Definicje schematów uprawnień
types.ts Współdzielone typy TypeScript
scripts Skrypty narzędziowe
init-permissions.ts Skrypt inicjalizujący system uprawnień
init_settings.ts Skrypt inicjalizujący ustawienia
seed.ts Skrypt zasilający bazę danych przykładowymi danymi
migrate_photos.ts Skrypt migracji zdjęć
public Pliki statyczne serwera
assets Zasoby statyczne (obrazy, ikony)
uploads Folder na przesyłane pliki
service-worker.js Service worker do obsługi PWA
backups Kopie zapasowe plików

Najważniejsze pliki konfiguracyjne

package.json Konfiguracja projektu i zależności
drizzle.config.ts Konfiguracja ORM Drizzle
postcss.config.js Konfiguracja PostCSS
tailwind.config.ts Konfiguracja Tailwind CSS
vite.config.ts Konfiguracja Vite (bundler)
tsconfig.json Konfiguracja TypeScript
